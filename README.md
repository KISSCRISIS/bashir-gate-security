# نظام أمن بوابة الطوارئ — مستشفى البشير

## الملفات
- `index.html` — شاشة الحارس (كود QR + حالة لحظية + زر البلاغ)
- `verify.html` — صفحة تحقق/تسجيل الكادر الطبي (تُفتح بمسح الكود)
- `login.html` — دخول المشرفين
- `admin_dashboard.html` — غرفة التحكم (طلبات، سجل، إدارة مشرفين، إنذار)
- `schema.sql` — كل أوامر قاعدة البيانات (نفّذه أولاً)
- `assets/style.css` + `assets/supabase-client.js` — مشتركة بين الصفحات

## خطوات التشغيل (بالترتيب)

**1. قاعدة البيانات**
افتح Supabase Dashboard ← SQL Editor ← نفّذ محتوى `schema.sql` كامل دفعة واحدة.

**2. إضافة أول حساب أدمن**
داخل `schema.sql` فيه سطر معلّق بهذا الشكل، فعّله بقيمك الحقيقية ونفّذه لحاله:
```sql
insert into admins (employee_id, phone, role) values ('EMP001', '0790000000', 'SUPER_ADMIN');
```

**3. النشر على Netlify**
اسحب مجلد المشروع كامل (لازم يضم `assets/` مع باقي الملفات بنفس المسار) على Netlify
(Drag & Drop أو ربط مستودع Git). لا حاجة لأي إعدادات Build، هذه ملفات ثابتة.

**4. الاختبار**
- افتح `index.html` على تابلت/جهاز الحارس.
- افتح `verify.html` من جوال آخر (بمسح الكود فعلياً أو بفتح الرابط مباشرة للتجربة).
- افتح `login.html` وسجّل دخول بالحساب اللي أضفته بالخطوة 2، جرّب لوحة التحكم.

**5. تذكير مهم بخصوص صوت الإنذار**
أول ما يفتح الأدمن `admin_dashboard.html`، لازم يضغط زرّ "🔔 تفعيل صوت الإنذار" مرة واحدة
(قيود المتصفحات على تشغيل الصوت تلقائياً بدون تفاعل من المستخدم). بعدها الإنذار يعمل تلقائياً
لباقي الجلسة عند أي بلاغ جديد.

## ملاحظات أمنية مهمة (اقرأها قبل الاستخدام الفعلي بمستشفى حقيقي)
- جدول `admins` **مقفول تماماً** لقراءة anon — الدخول يتم فقط عبر دالة `verify_admin_login` الآمنة، حتى لا يقدر أي زائر يجيب أرقام هواتف المشرفين من المتصفح مباشرة.
- جدول `pending_registrations` (فيه أسماء وهواتف الكادر) مفتوح بالقراءة العامة حالياً لأن `verify.html` و لوحة التحكم يحتاجونه — هذا مقبول لمرحلة تجريبية، لكن للاستخدام الفعلي الكامل يُفضّل الانتقال لـ Supabase Auth (تسجيل دخول حقيقي بـ OTP على الهاتف) وربط RLS بهوية المستخدم، خصوصاً لحماية `admin_dashboard.html` بشكل حقيقي على مستوى القاعدة وليس فقط بفحص الواجهة.
- زر "إبلاغ عن مخالفة" بشاشة الحارس بدون أي تسجيل دخول بالتصميم (Kiosk) — هذا مقصود حسب طلبك، لكن يعني أي شخص يقف أمام شاشة الحارس يقدر يضغطه. إذا حبيت تقييده مستقبلاً ممكن نضيف رمز PIN قصير قبل الإرسال.
